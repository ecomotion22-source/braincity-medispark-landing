커스텀 도메인을 연결할 예정이라면, 네이버 등록 전에 아래 항목을 새 도메인으로 다시 바꾸는 것을 권장합니다.

## 등록 절차

### 1. 네이버 서치어드바이저 접속

- 접속: [네이버 서치어드바이저](https://searchadvisor.naver.com/)
- 네이버 계정으로 로그인

### 2. 사이트 등록

- 웹마스터 도구에서 사이트 주소 입력
- 반드시 **호스트 단위**로 등록

예시:

- `https://example.com` 등록 가능
- `https://example.com/landing` 등록 불가

### 3. 사이트 소유 확인

네이버는 보통 아래 2가지 방식을 제공합니다.

1. HTML 태그 방식
2. HTML 파일 업로드 방식

이 프로젝트는 **HTML 태그 방식**이 가장 간단합니다.

#### HTML 태그 방식 적용 위치

`index.html` 안에 아래 메타 태그가 이미 들어가 있습니다.

```html
<meta name="naver-site-verification" content="REPLACE_WITH_NAVER_VERIFICATION_CODE" />
```

서치어드바이저가 발급한 실제 코드로 `REPLACE_WITH_NAVER_VERIFICATION_CODE` 부분만 바꾸고 배포하면 됩니다.

### 4. 사이트맵 제출

서치어드바이저에서 사이트맵 제출 메뉴로 이동한 뒤 아래 주소를 제출합니다.

```text
https://braincity-medispark-landing-git-main-ecomotion22-8919s-projects.vercel.app/sitemap.xml
```

### 5. robots.txt 확인

아래 주소가 열리는지 확인합니다.

```text
https://braincity-medispark-landing-git-main-ecomotion22-8919s-projects.vercel.app/robots.txt
```

### 6. 수집 요청

- 서치어드바이저에서 `웹페이지 수집` 또는 유사 메뉴로 메인 URL 요청
- 메인 URL:

```text
https://braincity-medispark-landing-git-main-ecomotion22-8919s-projects.vercel.app/
```

### 7. 반영 상태 확인

- 수집 현황
- 색인 현황
- 웹페이지 최적화 진단

을 순차적으로 점검합니다.

## 운영 체크리스트

- 메인 페이지가 `200 OK`로 열리는지 확인
- `/api/health`는 검색 대상이 아니므로 제출 불필요
- `robots.txt`에서 `Yeti` 접근 허용 확인
- 사이트맵 URL과 실제 도메인이 일치하는지 확인
- 소유확인 후 다시 한 번 수집 요청

## 참고 사항

- 단일 랜딩 페이지이므로 `RSS`는 필수는 아닙니다.
- 현재 구조에서는 `sitemap.xml`만으로 충분합니다.
- 도메인이 변경되면 사이트맵과 canonical도 같이 변경해야 합니다.
