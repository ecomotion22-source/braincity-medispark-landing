# 구글시트 앱스크립트 연동 설정 가이드

작성일: 2026-07-01

## 1. 목적

이 가이드는 현재 랜딩페이지의 상담문의 데이터를 Google Sheets에 자동 저장하기 위한 설정 방법을 설명한다.

완성 후 동작 흐름은 아래와 같다.

1. 사용자가 랜딩페이지에서 `관심고객 등록` 또는 `전화상담`을 진행한다.
2. 랜딩 서버가 고객정보를 로컬 파일에 저장한다.
3. 같은 데이터를 Google Apps Script 웹앱 URL로 POST 전송한다.
4. Apps Script가 Google Spreadsheet의 `상담문의` 시트에 한 줄씩 추가한다.

## 2. 현재 랜딩페이지에서 이미 준비된 것

현재 코드에는 아래 기능이 이미 들어가 있다.

- 메인 문의 폼 저장
- 전화상담 버튼 클릭 시 `이름 + 연락처` 입력 후 전화 연결
- 로컬 저장 파일: `data/leads.jsonl`
- Google Apps Script 웹앱 URL이 설정되면 자동 전송

관련 파일:

- 랜딩 서버: [server.js](C:\aptup-land\apt\server.js)
- 전화상담 UI: [index.html](C:\aptup-land\apt\index.html)
- 클라이언트 동작: [script.js](C:\aptup-land\apt\script.js)
- Apps Script 코드: [google-apps-script/Code.gs](C:\aptup-land\apt\google-apps-script\Code.gs)

## 3. Google Spreadsheet 만들기

1. Google Drive에서 새 스프레드시트를 만든다.
2. 파일명을 예를 들어 `브레인시티 메디스파크 상담문의`로 지정한다.
3. 시트 이름은 비워둬도 된다.

현재 Apps Script 코드는 `상담문의`라는 시트를 자동으로 만들거나, 이미 있으면 그 시트를 사용한다.

## 4. Apps Script 프로젝트 만들기

Google 공식 문서 기준으로, 스프레드시트 안에서 `확장 프로그램 > Apps Script`로 들어가면 해당 시트에 바인드된 스크립트 프로젝트를 만들 수 있다.  
참고: [Script Projects | Apps Script](https://developers.google.com/apps-script/guides/projects)

실행 순서:

1. 방금 만든 Google Spreadsheet를 연다.
2. 상단 메뉴에서 `확장 프로그램 > Apps Script`를 클릭한다.
3. 새 프로젝트가 열리면 기본 코드 내용을 모두 지운다.
4. 현재 폴더의 [google-apps-script/Code.gs](C:\aptup-land\apt\google-apps-script\Code.gs) 내용을 그대로 붙여넣는다.
5. 프로젝트 이름을 예를 들어 `분양랜딩_상담문의_수집`으로 저장한다.

## 5. Apps Script 코드 설명

핵심 함수는 `doPost(e)`다.

Google 공식 문서 기준으로 `doPost(e)`는 웹앱이 HTTP POST 요청을 받을 때 실행된다.  
참고: [Simple Triggers - doGet(e) and doPost(e)](https://developers.google.com/apps-script/guides/triggers)

현재 코드 동작:

- `e.postData.contents`에서 JSON 본문을 읽는다.
- 현재 스프레드시트의 `상담문의` 시트를 찾는다.
- 시트가 없으면 새로 만든다.
- 첫 줄이 비어 있으면 헤더를 만든다.
- 이후 한 줄씩 데이터를 추가한다.

Google Sheets에 행을 추가하는 방식은 `sheet.appendRow([...])`를 사용한다.  
참고: [Sheet.appendRow(rowContents)](https://developers.google.com/apps-script/reference/spreadsheet/sheet)

## 6. 웹앱으로 배포하기

Google 공식 문서 기준으로 Apps Script를 웹앱으로 배포하려면 `Deploy > New deployment > Web app` 순서로 진행한다.  
참고: [Web Apps | Apps Script](https://developers.google.com/apps-script/guides/web)

실행 순서:

1. Apps Script 편집기 우상단에서 `배포 > 새 배포`를 클릭한다.
2. `유형 선택`에서 `웹 앱`을 선택한다.
3. 설명은 예를 들어 `분양 랜딩 상담 문의 수집`으로 입력한다.
4. 실행 계정은 본인 계정으로 둔다.
5. 접근 권한은 `Anyone` 또는 `Anyone with the link` 성격의 공개 실행 가능 옵션으로 설정한다.
   이유:
   현재 랜딩 서버가 외부에서 HTTP POST로 호출해야 하기 때문이다.
6. `배포`를 클릭한다.
7. 처음 배포하면 권한 승인 화면이 나오므로 승인한다.
8. 배포가 끝나면 웹앱 URL을 복사한다.

웹앱 배포와 테스트 방식은 Google 공식 문서의 `Deploy a script as a web app`, `Test deployments` 절차를 따르면 된다.  
참고:

- [Web Apps | Apps Script](https://developers.google.com/apps-script/guides/web)
- [Create and manage deployments | Apps Script](https://developers.google.com/apps-script/concepts/deployments)

## 7. 랜딩 서버에 웹앱 URL 연결하기

현재 랜딩 서버는 환경변수 `GOOGLE_APPS_SCRIPT_WEB_APP_URL` 값을 읽어서, 문의 등록 시 Apps Script로 데이터를 전송한다.

PowerShell에서 실행:

```powershell
$env:GOOGLE_APPS_SCRIPT_WEB_APP_URL="https://script.google.com/macros/s/여기에_배포_ID/exec"
node server.js
```

이미 서버가 켜져 있다면:

1. 현재 Node 서버를 종료한다.
2. 위 환경변수를 설정한 뒤 다시 `node server.js`로 실행한다.

## 8. 구글시트에 저장되는 컬럼

현재 Apps Script는 아래 순서로 저장한다.

1. 등록시각
2. 이름
3. 연락처
4. 관심평형
5. 상담희망
6. 문의내용
7. 유입경로
8. 전화연결여부
9. 원격주소

예시:

- 메인 폼 등록
  - 유입경로: `메인문의폼`
  - 전화연결여부: `N`

- 전화상담 버튼 등록
  - 유입경로: `전화상담버튼`
  - 전화연결여부: `Y`

## 9. 실제 테스트 방법

### 9-1. Apps Script 단독 테스트

1. 웹앱 URL을 배포한다.
2. 랜딩 서버에 URL을 설정한다.
3. 랜딩페이지에서 테스트 문의를 1건 등록한다.
4. Google Spreadsheet를 열어 `상담문의` 시트에 행이 추가되었는지 확인한다.

### 9-2. 전화상담 흐름 테스트

1. 랜딩페이지 상단 또는 하단의 `전화상담` 버튼을 누른다.
2. 이름과 연락처를 입력한다.
3. 제출 후 전화 연결 시도가 되는지 확인한다.
4. 동시에 Google Sheets에 한 줄이 추가되는지 확인한다.

## 10. 자주 발생하는 문제

### 문제 1. 구글시트에 데이터가 안 들어간다

확인 순서:

1. Apps Script가 웹앱으로 배포되었는지 확인
2. 웹앱 URL이 `/exec` 형태인지 확인
3. `GOOGLE_APPS_SCRIPT_WEB_APP_URL` 환경변수가 정확한지 확인
4. Apps Script 권한 승인 완료 여부 확인
5. 새 배포 후 URL이 바뀌었는지 확인

### 문제 2. 시트는 있는데 값이 안 써진다

확인 순서:

1. Apps Script 코드가 [google-apps-script/Code.gs](C:\aptup-land\apt\google-apps-script\Code.gs)와 동일한지 확인
2. `doPost(e)`가 삭제되거나 이름이 바뀌지 않았는지 확인
3. Spreadsheet와 Script가 같은 문서에 바인드되어 있는지 확인

### 문제 3. 서버에서는 저장되는데 구글시트만 실패한다

현재 서버는 로컬 파일 저장과 구글시트 전송을 함께 시도한다.  
구글시트 전송이 실패해도 로컬 파일에는 기록이 남는다.

확인 파일:

- [data/leads.jsonl](C:\aptup-land\apt\data\leads.jsonl)

각 레코드 안의 `googleSync` 필드를 보면 전송 성공 여부를 확인할 수 있다.

## 11. 보안과 운영상 주의

### 공개 웹앱 특성

Apps Script 웹앱은 URL을 아는 요청자가 호출할 수 있으므로, 운영 단계에서는 최소한 아래 보강을 권장한다.

- 서버에서만 Apps Script를 호출하고 브라우저에서 직접 호출하지 않기
- Honeypot 유지
- 동일 IP 반복 호출 모니터링
- 필요하면 서버와 Apps Script 사이에 `secret` 값을 하나 더 넣어 검증하기

### 개인정보 관리

- 시트 접근 권한은 실제 상담 운영자만 부여
- 테스트 데이터는 운영 전에 삭제
- 추가로 상담상태, 담당자, 콜백여부 컬럼을 만들어 운영 가능
